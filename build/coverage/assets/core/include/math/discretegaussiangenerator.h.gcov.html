      37             : /**
      38             :  * This is the header file for DiscreteGaussianGenerator class, which contains 3
      39             :  * different sampling methods.
      40             :  *
      41             :  * First sampling method implemented is the rejection sampling defined in
      42             :  * section 4.1 of https://eprint.iacr.org/2007/432.pdf. It is usable for
      43             :  * arbitrary centers and standard deviations, and does not require any form of
      44             :  * precomputation. However, it has high rejection rates and is prone to timing
      45             :  * attacks. It is not used anywhere in the library at the moment and is here for
      46             :  * historical reasons.
      47             :  *
      48             :  * Second sampling method implemented is Karney's method defined in Algorithm D
      49             :  * from https://arxiv.org/pdf/1303.6257.pdf, which is an improved method based
      50             :  * on rejection sampling. It also works for arbitrary centers and standard
      51             :  * deviations without any precomputation. Its rejection rate is smaller than in
      52             :  * the rejection sampling method but it may still be vulnerable to timing
      53             :  * attacks.
      54             :  *
      55             :  *
      56             :  * Final sampling method defined in this class is the Peikert's inversion method
      57             :  * discussed in section 4.1 of https://eprint.iacr.org/2010/088.pdf and
      58             :  * summarized in section 3.2.2 of
      59             :  * https://link.springer.com/content/pdf/10.1007%2Fs00200-014-0218-3.pdf. It
      60             :  * requires CDF tables of probabilities centered around single center to be
      61             :  * kept, which are precalculated in constructor. The method is not prone to
      62             :  * timing attacks but it is usable for single center, single deviation only.
      63             :  * It should be also noted that the memory requirement grows with the standard
      64             :  * deviation, therefore it is advised to use it with smaller deviations.   */
